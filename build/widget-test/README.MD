To test the bchat embeddable widget on your local test site, you have two primary methods to integrate the embed code generated by the Agent Admin UI into your HTML pages (such as [index.html](file:///home/chaschel/Documents/go/bchat/widget/site/index.html)).

---

### Method 1: Direct Integration with Local Backend (Recommended)
This method loads `embed.js` dynamically from your active Go backend. Any updates you make to the backend handlers or properties will reflect immediately.

1. In your test page (e.g., [index.html](file:///home/chaschel/Documents/go/bchat/widget/site/index.html)), replace the widget script tags at the bottom of the body (around line 318) with the snippet from the Agent Admin UI:

```html
<!-- Load the widget code dynamically from your bchat backend -->
<script src="http://localhost:8081/widget/<YOUR_TENANT_SLUG>/embed.js"></script>
<script>
  AgentChatWidget.init({
    tenant: "<YOUR_TENANT_SLUG>",
    baseUrl: "http://localhost:8081",
    color: "#0d9488",                // Customize colors
    position: "bottom-right",        // "bottom-right" or "bottom-left"
    welcomeMessage: "Hi! How can I help you today?",
    companyName: "<YOUR_COMPANY_NAME>"
  });
</script>
```
*Replace `<YOUR_TENANT_SLUG>` with your actual tenant slug (e.g., `rizal` or `scraper`) and configure other parameters accordingly.*

---

### Method 2: Static Integration (Using compiled static bundle)
This method uses a statically copied version of `embed.min.js` served directly by the Python HTTP server.

1. Build the widget bundle:
   ```bash
   cd widget
   npm install && npm run build
   ```
2. Copy the compiled asset to the static site folder:
   ```bash
   cp dist/embed.min.js site/
   ```
3. Update [index.html](file:///home/chaschel/Documents/go/bchat/widget/site/index.html)'s embed block (lines 318–328) to point to the local `embed.min.js` and initialize with your tenant parameters:

```html
<!-- Load the compiled static file -->
<script src="embed.min.js"></script>
<script>
  AgentChatWidget.init({
    tenant: "<YOUR_TENANT_SLUG>",
    baseUrl: "http://localhost:8081",
    color: "#0d9488",
    position: "bottom-right",
    welcomeMessage: "Hi! How can I help you today?",
    companyName: "<YOUR_COMPANY_NAME>"
  });
</script>
```

---

### Method 3: Iframe Embedding (DOM Isolation)
If you are testing how the widget performs isolated inside an `<iframe>` (to avoid CSS/JS conflicts), you can embed the iframe code directly:

1. Insert the `<iframe>` tag anywhere inside your page body (refer to [iframe-test.html](file:///home/chaschel/Documents/go/bchat/widget/site/iframe-test.html)):

```html
<iframe
  src="http://localhost:8081/widget/<YOUR_TENANT_SLUG>/iframe?color=%230d9488&welcome=Hello%21%20How%20can%20we%20help%3F&companyName=<YOUR_COMPANY_NAME>"
  style="position:fixed;bottom:20px;right:20px;width:350px;height:500px;border:none;z-index:9999;border-radius:12px;box-shadow:0 8px 32px rgba(0,0,0,0.15);"
  title="Chat Widget"
></iframe>
```

### How the Backend Serves the Widget Assets
- **Javascript Embed (`/widget/:slug/embed.js`)**: Handled by [HandleWidgetEmbed](file:///home/chaschel/Documents/go/bchat/server/router/api/v1/agent/handlers.go#L1960). It looks for `widget/dist/embed.min.js`. If found, it prepends the `window.AgentChatConfig` configuration block and serves it. If the file is missing, it falls back to dynamically rendering a basic client-side JS script via [generateWidgetScript](file:///home/chaschel/Documents/go/bchat/server/router/api/v1/agent/handlers.go#L1698).
- **Iframe Embed (`/widget/:slug/iframe`)**: Handled by [HandleWidgetIframe](file:///home/chaschel/Documents/go/bchat/server/router/api/v1/agent/handlers.go#L2015). It dynamically returns a fully rendered wrapper page that automatically opens the widget inside the iframe layout.
