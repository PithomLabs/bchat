# Bug Report: Missing `version` Column in `agent_source_files`

**Date:** 2026-01-30 22:04
**Status:** Fixed
**Severity:** High (Production blocking)

---

## Summary

After deploying to fly.io, uploading KB.MD files failed with:
```
ERROR failed to save KB source file error="failed to get next version: SQL logic error: no such column: version (1)"
```

## Root Cause

Migration `12__agent_source_file_version.sql` created an INDEX on the `version` column but **never added the column** to the table:

```sql
-- Broken migration 12 (only creates index, doesn't add column)
CREATE INDEX IF NOT EXISTS idx_source_files_version ON agent_source_files(tenant_id, audience_type, file_type, version DESC);
```

The Go code at `store/db/sqlite/agent.go:1033-1043` expects the `version` column to exist for versioning KB/Policy files.

### Why This Wasn't Caught

1. **Local dev database had column manually added** - Developers may have run ad-hoc SQL to add the column during development
2. **No schema validation tests** - No automated tests verified that migrations produce the expected schema
3. **No CRUD smoke tests** - No tests verified that INSERT/SELECT operations work after migrations
4. **Fresh production database exposed the bug** - fly.io ran migrations on a fresh database

## Fix Applied

### 1. Immediate Fix - Migration 20

Created `store/migration/sqlite/0.25/20__agent_source_file_version_fix.sql`:

```sql
-- Fix: Add version column that was missing from migration 12
ALTER TABLE agent_source_files ADD COLUMN version INTEGER NOT NULL DEFAULT 1;
```

### 2. Long-term Prevention - Schema Validation System

#### New Test Files

| File | Purpose |
|------|---------|
| `store/test/schema_validation_test.go` | Verifies migrations create expected schema |
| `store/test/agent_crud_test.go` | CRUD smoke tests for agent tables |

#### Schema Validation Utilities

Added to `store/migration_helper.go`:

- `GetTableColumns(ctx, db, tableName)` - Returns list of column names for a table
- `ValidateTableSchema(ctx, db, tableName, requiredColumns)` - Verifies required columns exist

#### Taskfile Command

Added `validate:schema` task to Taskfile.yml:

```bash
task validate:schema
```

Runs schema validation and CRUD smoke tests before deployment.

## Prevention Checklist

For future migrations:

- [ ] Test migration on **fresh database** (not just dev with existing schema)
- [ ] Add column BEFORE creating index on it
- [ ] Run `task validate:schema` before committing
- [ ] Verify Go code column references match migration schema
- [ ] Add corresponding test cases for new tables/columns

## Lessons Learned

1. **Always test migrations on fresh databases** - Local dev databases accumulate manual changes
2. **Schema validation tests are essential** - They catch migration bugs before production
3. **CRUD smoke tests catch runtime issues** - Schema may apply but operations may fail
4. **Pre-deployment validation prevents downtime** - `task validate:schema` should run in CI/CD
